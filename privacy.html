Summary of data access

We request only the Google scopes required to deliver core functionality: composing and sending messages, reading message content and metadata while the add-on UI is active, and temporary cloud storage for processing exports and attachments. Access is limited to user-authorized sessions and contextual UI actions.

How we use data

All operations are initiated by the user. Compose and send are used to create and transmit messages; read and metadata access are used to display message previews and allow selection; cloud storage is used only for transient exports or attachment processing during user-triggered jobs.

Retention and deletion

We do not retain email content long-term. Temporary files generated for processing are deleted automatically upon job completion. If a user explicitly exports or saves data, retention is determined by the user's chosen destination.

User controls and safeguards

Every destructive or bulk action requires explicit confirmation and shows an item count and sample. Long operations include progress, cancel capability, configurable batch size limits, and undo where feasible. No action is taken silently.

Security

Tokens and credentials are stored securely and scoped minimally. Processing happens in isolated jobs with least-privilege access and encrypted transport. We follow standard security best practices for data handling.
